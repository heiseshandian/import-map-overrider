🚀 Import Map Overrider 测试页面

这个页面包含了多个 Import Maps，用于测试扩展的功能

📋 使用说明

如何测试扩展（Service Worker 模式）：

  1. 确保已安装 Import Map Overrider 扩展
  2. 点击浏览器工具栏中的扩展图标
  3. 查看检测到的 Import Maps
  4. 添加覆盖规则（例如：react → https://cdn.skypack.dev/react@17.0.2）
  5. 刷新页面使 Service Worker 拦截规则生效
  6. 使用下面的测试按钮验证覆盖是否生效
⚠️ 重要提示：扩展现在使用 Service Worker 拦截网络请求来实现覆盖功能。添加或修改覆盖规则后，必须刷新页面才能看到效果。

💡 测试建议

推荐的测试流程：

  1. 查看原始状态：点击"显示当前 Import Maps"查看初始状态
  2. 测试导入：点击"测试 React 导入"等按钮，观察导入的模块信息
  3. 添加覆盖：使用扩展添加覆盖规则，例如：
    • 包名：react
    • 新 URL：https://cdn.skypack.dev/react@17.0.2
  4. 刷新页面：添加覆盖规则后，刷新页面使 Service Worker 拦截生效
  5. 验证覆盖：再次点击"测试 React 导入"，检查是否使用了新的 URL
  6. 查看变化：点击"显示当前 Import Maps"查看原始状态（Service Worker 在网络层拦截，不会修改页面的 Import Maps）

💡 工作原理：扩展现在使用 Service Worker 模式拦截网络请求并重定向到新的 URL，这比之前的 Import Map 注入方式更加稳定可靠。

🔗 智能依赖：扩展会自动处理相关依赖包的版本兼容性，如将 Vue 3.5.16 与兼容的 vue-demi 版本自动匹配。

🐛 常见问题测试案例

Vue 版本升级依赖错误

问题描述：将 Vue 从 3.5.13 升级到 3.5.16 后出现错误：

Uncaught SyntaxError: The requested module '/npm/vue-demi@0.14.6/+esm' does not provide an export named 'set'

解决方案：

  1. 在扩展中添加 Vue 覆盖规则：
    包名：vue
    新 URL：https://cdn.jsdelivr.net/npm/vue@3.5.16/dist/vue.esm-browser.prod.min.js
  2. 扩展会自动将 vue-demi 映射到兼容版本 vue-demi@0.14.10
  3. 刷新页面验证修复效果

💡 原理：不同版本的 Vue 需要不同版本的 vue-demi 依赖包，扩展的智能依赖映射功能会自动处理这种兼容性问题。

Vue Router 导出错误

问题描述：使用 Vue 3.x 时出现 vue-router 导出错误：

Uncaught SyntaxError: The requested module 'vue-router' does not provide an export named 'useRouter'

解决方案：

  1. 自动解决：当您添加 Vue 覆盖规则时，扩展会自动映射兼容的 vue-router 版本
  2. 手动添加：如果需要特定版本，可手动添加：
    包名：vue-router
    新 URL：https://cdn.jsdelivr.net/npm/vue-router@4.4.5/dist/vue-router.esm-browser.js
  3. 刷新页面验证修复效果

💡 原理：Vue 3.x 需要使用 vue-router 4.x 版本，且必须使用 ESM 浏览器构建版本才能正确导出 useRouter 等组合式 API。
