üêõ Common Issue Test Cases

Vue Version Upgrade Dependency Error

Problem Description: Error occurs after upgrading Vue from 3.5.13 to 3.5.16:

Uncaught SyntaxError: The requested module '/npm/vue-demi@0.14.6/+esm' does not provide an export named 'set'

Solution:

  1. Add Vue override rule in the extension:
    Package Name:vue
    New URL:https://cdn.jsdelivr.net/npm/vue@3.5.16/dist/vue.esm-browser.prod.min.js
  2. The extension will automatically map vue-demi to compatible version vue-demi@0.14.10
  3. Refresh the page to verify the fix

üí° Principle: Different versions of Vue require different versions of vue-demi dependency packages. The extension's intelligent dependency mapping feature automatically handles such compatibility issues.

Vue Router Export Error

Problem Description: Vue-router export error occurs when using Vue 3.x:

Uncaught SyntaxError: The requested module 'vue-router' does not provide an export named 'useRouter'

Solution:

  1. Automatic resolution: When you add Vue override rules, the extension will automatically map compatible vue-router versions
  2. Manual addition: If you need a specific version, you can manually add:
    Package Name:vue-router
    New URL:https://cdn.jsdelivr.net/npm/vue-router@4.4.5/dist/vue-router.esm-browser.js
  3. Refresh the page to verify the fix

üí° Principle: Vue 3.x requires vue-router 4.x version, and must use the ESM browser build version to correctly export useRouter and other composition APIs.
